# ProductScanner 
ProductScanner - Android-приложение для сканирования QR/штрих-кодов продуктов питания с последующим анализом состава, выявлением аллергенов и формированием персонализированных рекомендаций.

# Описание проекта
Приложение позволяет пользователю:
- Сканировать штрих-коды и QR-коды продуктов.
- Получать подробную информацию о составе продукта через API глобальной базы данных OpenFoodFacts.
- Проверять состав на наличие аллергенов и других индивидуальных ограничений пользователя.
- Оценивать продукты и получать персонализированные рекомендации на основе оценок.
- Приложение актуально для людей с пищевыми ограничениями, спортсменов, родителей маленьких детей, вегетарианцев и всех, кто следит за питанием.

# Технологический стек
- Kotlin — основной язык разработки Android-приложения.
- Jetpack Compose — создание адаптивного и современного пользовательского интерфейса.
- CameraX & ZXing — для работы с камерой и распознавания штрих/QR-кодов.
- SQLite + Room — локальное хранение данных (история сканирования, рейтинги).
- Firebase (Firestore + Authentication) — облачное хранение данных и аутентификация пользователей.
- Python + Flask — серверная часть для интеграции машинного обучения и API чат-бота.
- Scikit-learn (Python) — модуль машинного обучения для формирования персонализированных рекомендаций.

# Архитектура
Приложение построено по паттерну MVVM (Model-View-ViewModel):  
- Model — логика обработки данных (API-запросы, анализ состава, обработка ограничений ).
- View — пользовательский интерфейс с помощью Jetpack Compose.
- ViewModel — связывает UI и логику приложения, управляет состоянием экрана.

База данных содержит следующие таблицы:
- users — данные пользователей.
- products — информация о продуктах.
- scanned_products — история сканирования.
- user_restrictions — ограничения пользователя.
- user_product_rating — оценки продуктов.
- user_recommendation — рекомендации пользователю.

# Функциональные модули
Сканирование продуктов:
- Камера устройства и распознавание QR/штрих-кодов.
- Обработка изображения и получение кода.

Получение состава продукта:
- Запрос к OpenFoodFacts API.
- Передача состава на обработку чат-боту для расшифровки.

Анализ ограничений:
- Проверка состава продукта на наличие индивидуальных ограничений.

Рекомендательная система
- Формирование рекомендаций с использованием TF-IDF векторизации и косинусной близости.
